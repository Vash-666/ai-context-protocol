## 📝 **Lessons Learned**

### **What Works:**
1. **Small company model** - Core consciousness + departments
2. **Quality Equation focus** - 65% prompt files, 20% memory, 10% model, 5% tools
3. **Three-tier switching** - Session continuity is critical
4. **Agent consolidation** - Fewer, more capable agents > many overlapping
5. **Automation first** - Health monitoring, GitHub, cleanup

### **What to Avoid:**
1. **Project fragmentation** - Keep all projects under /projects/
2. **Agent sprawl** - Consolidate overlapping capabilities
3. **Manual processes** - Automate everything possible
4. **Hype language** - Focus on truth density and facts
5. **Complex computations** - Chunk tasks, pre-compute heavy operations

### **Key Insights:**
1. **Quality In → Quality Out** - Invest in prompt files (65% impact)
2. **Context is everything** - Model switching loses context without protocol
3. **Automation pays dividends** - Time saved compounds
4. **Simple > Complex** - Chunked tasks succeed, monolithic fail
5. **Truth > Hype** - Recruiters value real work over exaggeration
